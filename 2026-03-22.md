# 2026-03-22

## 故障修复：切换模型导致会话崩溃

**时间：** 17:21

**现象：** 在 Control UI 切换模型后，会话崩溃，报错 `No API key found for provider "zhipu"`

**原因：** auth-profiles.json 里没有 zhipu 的 API key 配置

**修复：** 用 Claude Code 添加了 zhipu API key 到 auth-profiles.json，重启 Gateway

**教训：** 切换模型前，确保该 provider 的 API key 已经在 `~/.openclaw/agents/main/agent/auth-profiles.json` 中配置好

**命令参考：**
```bash
# 查看当前配置的 auth profiles
cat ~/.openclaw/agents/main/agent/auth-profiles.json

# 添加 API key（通过 CLI）
openclaw agents add main
```
